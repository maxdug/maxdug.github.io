LAB | Learning at BCG

Boston Consulting Group is one of the "Big Three" consultancies and the second largest by revenue in the world. In 2019, I joined LAB, BCG’s internal learning team as they transitioned from an in-house learning managment system to a third-party platform.

My role was to lead UX design, engage stakeholders, and facilitate front-end development.

Guiding stakeholders

Leading a team consisting of a BA, PM, and designer - I helped stakeholders bring their content over to the new platform. These ranged from subsidiary businesses BCG's internal professors. The challenge was to onboard them to an unfamiliar platform and process.

Kickoff Communication

During kickoffs, I walked stakeholders through a timelines, requirements, personas basics and other areas to onboard to set expectations for what a full design project entails

Explaining Zoomed Out Template Flow

Using a zoomed out user flow to explain the templates and how the work together.

Research and Discovery

Once in discovery, I would analyse the existing experience and reach out to users to explore opportunities.

Mapping the Existing Experience

The first step in analysis of the new learning platform was to map the experience end-to-end. I then breaking out discrete flows for further details.

Overlaying Data

We layered qualitative data onto the user flows to show context and help stakeholders prioritize improvements.

Feedback, Research, and Initatives in Harmony

During my time, the learning team started a dedicated transitioned to Agile and I was tasked with developing a model for converting all our user feedback into intiatives to work on and research. To solve this problem, I created a multi-step process of sorting the feedback and then applying existing "Critical to Quality" and "Kano" models.

Card Sorting During the Pandemic

Analytics identified that learners struggled navigating our catalog. In the midst of the pandemic, we used a note app (Workflowy) to run virtual card sorting. This allowed a streamlined interface, where we could duplicate and save each configuration. We discovered that our catalog was based on an non-intuative business structure.

User Interviews

We followed card sorting by building a prototype and running moderated user interviews. For the virtual interviews, I developed scripts that ran on a seperate laptop to keep my connection with the user seamless. Users responded to titles that matched areas for improvement in their quarterly reviews.

Rapid Prototypes = Rapid Results

During prototyping, we used our feedback model to overlay data, problems, and solutions. This helped demo new designs to stakeholders and back it up with researh. Our biggest challenge was to provide a streamlined, personalized experience for learners, despite some confusing, fixed UI in the learning management system (LMS). Our solution was to focus on entry points and side-steped the LMS, instead of competing. We created a new landing and micro-site pages that guide learners through personalized learning journeys.

Visual Design That’s Not From Scratch

A modular system to last. After analyzing the card sorting results, we iterated on designs for a memamenu and setup prototypes with Figma to facilitate more testing.

Getting my hands dirty with development

The platform which we developed, provided a unique set of challenges with the chief among them being a restriction in the code we could use. When thinking about developing a UI pattern library to pull from, we only had HTML & CSS to work with. Through going back to basics, we were able to provide customized, interactive UI with clean CSS.

We also had a unique challenge where we couldn’t use media queries - the standard way to modify a website to be responsive. So this project became a masterclass in using flexbox to create modules that worked across all breakpoints.

I was brought into BCG as a UX designer with the understanding that they needed some light development work as well.
